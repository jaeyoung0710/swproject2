# 마이크로 컨트롤러와 마이크로 프로세서의 근본적인 차이
- 마이크로 컨트롤러는 CPU, RAM, Flash 메모리, GPIO, 타이머 등의 주변장치를 하나의 칩에 모두 내장한 장치이다. 
- 마이크로 프로세서는 CPU만 포함하고 있으며, RAM, 저장장치 등을 외부에 연결해야만 동작한다.

# 피지컬어드레스와 버추얼 어드레스의 구분
- 피지컬어드레스는 실제 RAM의 메모리 주소이다
- 버추얼 어드레스는 프로그램이 사용하는 가상의 메모리 주소이다

# super영역과 user 영역의 구분
- supuer 영역은 운영체제가 사용하는 관리자 권한 영역이다 또한 모든 하드웨어와 메모리에 접근이 가능하다
- User 영역은 일반 프로그램이 사용하는 영역이고 허가되지 않은 메모리나 장치에는 접근할 수 없다

# MCU에는 없지만 MPU에는 있는것
: MMU, 가상 메모리, 캐시, 운영체제 실행기능등이 있다. 반면 MCU는 이러한 기능 대신 저전력과 실시간 제어에 집중하도록 설계되어있다.

# MMU의 역할
- MMU는 메모리를 효율적으로 관리하는 장치로 가상 주소를 물리 주소로 전환, 메모리 보호, 프로세스 간 메모리 분리, 운영체제의 안정성 향상같은 역할을 한다.

# 자동차에서 MCU가 많이 사용되는 이유는?
- 자동차의 전자 제어 장치는 실시간으로 빠르게 동작해야하기 때문에 실시간 응답이 빠르고 소비전력인 낮고 가격이 저렴하고 센서와 액추에이터를 직접 제어 가능한 MCU가 많이 사용된다

# MCU의 정해진 역할
- 자동차 ECU 제어
- 에어백 제어
- ABS 제어
- 세탁기 제어 등등

# 펌웨어를 64BiT 마이크로프로세서에서도 사용할 수 있는가?
- 사용할 수 있다 다만 64bit 마이크로 프로세서에서는 펌웨어만으로 동작하는 경우는 드물다. 일반적으로 다른 운영체제를 먼저 실행하기 위한 부트 펌웨어가 사용되고 이후 다양한 운영체제가 실행되어 다양한 프로그램을 관리한다.
- 32bit에서도 펌웨어사용이 가능하다
