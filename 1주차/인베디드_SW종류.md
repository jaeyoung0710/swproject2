# 펌웨어란?
- 펌웨어는 하드웨어와 소프트웨어를 연결하는 프로그램이며 마이크로 컨트롤러가 전원을 키면 가장 먼저 실행되는 소프트웨어입니다.

# 펌웨어를 저장하는 장치
- 펌웨어는 보통 flash 메모리에 저장합니다. 왜냐하면 전원이 꺼져도 데이터가 지워지지 않고 프로그램을 다시 수정하여 업로드 할 수 있고 읽기 속도가 빠르고 내구성이 우수하기 때문입니다.

# 펌웨어와 RTOS의 차이점?
- 펌웨어
    - 운영체제가 없다
    - 하나의 프로그램이 순차적으로 실행된다
    - 메모리 사용량이 적음
    - 단순한 장치 제어에 적합
- RTOS
    - 실시간 운영체제이다.
    - 여러 작업을 동시에 수행한다
    - 우선 순위에 따라 작업을 관리한다
    - 자동차, 드론, 산업용 로봇 등에 사용된다

# 펌웨어로 구현 할 수 없는 기능?
- 펌웨어만으로는 복잡한 멀티태스킹, 여러 프로그램 동시 실행, 작업 우선순위 자동관리, 메모리 보호 등등은 할 수 없다.

# RTOS의 정의 
: RTOS는 정해진 시간안에 작업을 수행하도록 설계된 실시간 운영체제입니다.

# 우선순위 역전?
- 낮은 우선순위의 작업이 중요한 자원을 먼저 사용하고 있어서 높은 우선 순위의 작업이 기다리는 현상을 말합니다.

# DeadLock이 발생하는 원인
- 자원의 상호 배제
- 여러 자원을 동시에 요청
- 자원을 선점할 수 없음
- 순환 대기

# 가장 쉬운 해결 방법?
- 자원을 항상 같은 순서로 획득한다.
- Mutex 사용 시간을 최소화한다.
- 필요없는 자원은 즉시 반환

# 스케줄러와 일반 프로그램의 차이점은?
- 스케줄러 
    - 운영체제의 일부이다.
    - 어떤 Task를 언제 실행할 지 결정한다.
    - CPu를 효율적으로 분배한다
- 일반 프로그램
    - 스케줄러의 제어를 받으며 실행된다
    - 스스로 CPU 관리하지 않는다.

# 스케줄러의 용도
- Task 생성 및 삭제
- CPU 할당
- 우선순위 관리
- Context Switching
- 실시간 작업수행
- 시스템 성능 향상
